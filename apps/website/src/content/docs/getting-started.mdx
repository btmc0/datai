jump is a browser-based session manager for AI agents, test runners, and long-running processes. Long-running command-line work is easy to start and annoying to supervise: agents, watchers, builds, and shells end up buried in tabs or tmux panes. jump gives you a live sidebar of everything running across a machine, visible from any browser.

## Install

This fork does not currently publish Homebrew or install-script releases. Build the binaries from the GitHub repo for now:

```bash
git clone https://github.com/sting8k/jump.git
cd jump
TMPDIR=/tmp GOWORK=$PWD/go.work go build -o ~/.local/bin/jump ./cli/jump/cmd/jump
TMPDIR=/tmp GOWORK=$PWD/go.work go build -o ~/.local/bin/jumpd ./services/jumpd/cmd/jumpd
```

When release automation is restored, binaries will be published through [GitHub Releases](https://github.com/sting8k/jump/releases). On Windows, use WSL.

## Run

```bash
jump pi              # launch a coding agent
jump pytest --watch  # or any command

jump                 # open the dashboard
```

`jump` auto-starts the daemon (`jumpd`) on first run. Running `jump` with no arguments opens the dashboard in a dedicated browser window (Chrome/Chromium `--app` mode when available, otherwise your default browser).

The first time, the sidebar will be empty. Click **Add project** to pick which project directories appear. Once added, sessions in those directories show up automatically. Click a session to attach a live terminal.

See [Using the UI](/using-the-ui) for a full walkthrough.
